# Dev Log — 2026-07-03

## runway-trends: Runway Trends page

**Goal**: Aggregate the user's saved runway reports (`RunwayReport` snapshots) into a trend view, placed under Analysis → Financial Planning.

### Delivered
- **Backend**: `RunwayTrendDTO`; `RunwayReportService.getTrend(familyId)` parses each `snapshotJson` server-side, extracts `liquidTotal/monthlyBurn/runwayMonths/depletionDate` into points (ascending by `savedAt`, corrupt JSON skipped), and enriches the latest + previous report's category breakdown from `ExpenseCategoryMajor` (`categories` / `previousCategories`); new `GET /runway/reports/trend`. Response never includes the raw `snapshotJson`.
- **Frontend**: `utils/runwayTrend.js` (pure functions: KPI / prior-report delta / range truncation / category rows); `views/analysis/RunwayTrend.vue` (4 KPI cards with vs-previous-report deltas, Chart.js switchable-metric trend chart, category expense table, empty/single-report states); `getRunwayTrend` added to `api/runway.js`; route `/analysis/runway-trend`; nav item under Financial Planning in Sidebar.

### Decisions
- Data source = aggregate saved snapshots (no historical recompute); one point per report.
- Per-report wording throughout (last-N reports / report value / vs last report), not the design mock's monthly wording.
- Snapshot amounts are already converted to USD at save time → the trend endpoint does no currency conversion (no `ExchangeRateService`).
- Category colors/names come from `ExpenseCategoryMajor` (existing color/name fields) — no hand-rolled palette.

### Process
- Ran /opsx:explore → propose → apply. TDD throughout, RED→GREEN each step.
- Mid-way found "category vs-previous delta needs the prior report's data" — the group-1 DTO didn't carry it → added `previousCategories` (its own RED→GREEN cycle).
- Two evaluator subagents: group 1 = 100/100, group 2 = 99/100.
- Backend 14 tests, frontend 24 tests all green. The Expense domain's 3 failures + 15 errors are pre-existing (reproduced at HEAD~2, not introduced here).

### Gotcha
- During smoke test, `GET /api/runway/reports/trend` returned 400 `Failed to convert 'id' with value: 'trend'` — the running backend was a stale build, so `/trend` fell through to `@GetMapping("/{id}")`. Restarting via `./backend/start.sh` fixed it. Recorded in CLAUDE.md.
