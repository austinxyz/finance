# Dev Log — 2026-07-03

## runway-trends：资金跑道趋势页

**目标**：把已保存的资金跑道报告（`RunwayReport` 快照）汇总成趋势视图，放在 分析 → 财务规划。

### 交付
- **后端**：`RunwayTrendDTO`；`RunwayReportService.getTrend(familyId)` 服务端解析每份 `snapshotJson`，抽取 `liquidTotal/monthlyBurn/runwayMonths/depletionDate` 成点（按 `savedAt` 升序，坏 JSON 跳过），并用 `ExpenseCategoryMajor` 富化最新与次新报告的分类明细（`categories` / `previousCategories`）；新增 `GET /runway/reports/trend`。响应不含原始 `snapshotJson`。
- **前端**：`utils/runwayTrend.js`（KPI/环比/区间截取/分类行 纯函数）；`views/analysis/RunwayTrend.vue`（4 KPI 卡含较上次报告环比、Chart.js 可切换指标趋势图、分类支出明细表、空/单报告态）；`api/runway.js` 加 `getRunwayTrend`；路由 `/analysis/runway-trend`；Sidebar 财务规划 加导航项。

### 决策
- 数据源 = 聚合已保存快照（不重算历史）；逐报告原始点。
- 文案按逐报告真实语义（近N份 / 报告值 / 较上次报告），不沿用设计稿按月措辞。
- 快照金额保存时已折算 USD → 趋势端点不涉及汇率转换（不注入 ExchangeRateService）。
- 分类颜色/名称取 `ExpenseCategoryMajor`（已有 color/name 字段），无需手写调色板。
