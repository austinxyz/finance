## ADDED Requirements

### Requirement: CLI-initialized component configuration
The project SHALL have a `components.json` at the frontend root, generated by the shadcn-vue CLI, defining the component library style, path aliases, and Tailwind config location.

#### Scenario: components.json present after init
- **WHEN** `npx shadcn-vue@latest init` is run
- **THEN** `frontend/components.json` exists with correct path aliases pointing to `src/`

#### Scenario: CSS design tokens in index.css
- **WHEN** shadcn-vue is initialized
- **THEN** `frontend/src/index.css` contains CSS custom properties (`--background`, `--foreground`, `--primary`, etc.) under `:root`

---

### Requirement: Existing UI components replaced by CLI-managed equivalents
The 5 existing manually-written components in `src/components/ui/` SHALL be replaced by shadcn-vue CLI-generated versions: `Button`, `Card`, `CardContent`, `CardHeader`, `CardTitle`.

#### Scenario: Button renders with variant prop
- **WHEN** `<Button variant="outline">Click</Button>` is rendered
- **THEN** the button displays with outline styling (border, no fill)

#### Scenario: Card composes correctly
- **WHEN** a `Card` wraps `CardHeader`, `CardTitle`, and `CardContent`
- **THEN** the composed layout renders without errors and displays child content

#### Scenario: Existing consumer unaffected
- **WHEN** `LiabilityDetail.vue` (the existing consumer of Card components) is loaded
- **THEN** the page renders correctly with no console errors

---

### Requirement: Additional UI components available
The following shadcn-vue components SHALL be added via CLI and available for use in future features: `Badge`, `Select`, `Dialog`, `Table`, `Tabs`, `Separator`, `Input`, `Label`, `Tooltip`.

#### Scenario: Badge renders with variant
- **WHEN** `<Badge variant="secondary">New</Badge>` is rendered
- **THEN** the badge displays with secondary styling

#### Scenario: Input renders with label
- **WHEN** `<Label for="email">Email</Label><Input id="email" />` is rendered
- **THEN** clicking the label focuses the input

#### Scenario: Dialog opens and closes
- **WHEN** a trigger button inside `DialogTrigger` is clicked
- **THEN** the `DialogContent` becomes visible
- **AND** clicking outside or pressing Escape closes it

---

### Requirement: Tailwind config includes shadcn-vue animation and content paths
The `tailwind.config.cjs` SHALL include `tailwindcss-animate` plugin and reference the correct `content` paths so shadcn-vue component class names are not purged in production builds.

#### Scenario: Animation classes available
- **WHEN** a shadcn-vue Dialog opens
- **THEN** entrance/exit animations render (fade-in, slide-in) via `tailwindcss-animate`

#### Scenario: Component classes not purged
- **WHEN** a production build is run (`npm run build`)
- **THEN** all shadcn-vue component utility classes are present in the output CSS
